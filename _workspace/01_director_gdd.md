## 1. 개요

- **타이틀**: 크리스탈 가드 (Crystal Guard)
- **컨셉 한 줄**: 골짜기 끝의 마법 수정을 노리고 몰려오는 몬스터 무리를, 네 종류의 마법 타워를 조합해 10번의 웨이브 동안 막아내는 클래식 타워 디펜스.
- **플랫폼**: HTML5 Canvas 웹 게임. 바닐라 JS ES 모듈, 빌드 도구 없음. `python3 -m http.server 8000`으로 실행. **(v2)** GitHub Pages 배포(`https://revfactory.github.io/tower-defense/`) 및 모바일 터치 지원 — §12.4, §12.5.
- **한 판 길이**: 10~15분.
- **왜 재미있는가**: "이번 웨이브를 버틸 최소 투자 vs 다음 웨이브를 위한 저축"이라는 매 순간의 경제 선택이, 보스가 등장하는 10웨이브를 향해 긴장을 누적시킨다.

## 2. 코어 루프

```
[준비] 골드로 타워 건설/업그레이드
   ↓
[웨이브 시작] (버튼 또는 카운트다운)
   ↓
[전투] 타워 자동 공격 — 적 처치 시 골드, 누수 시 라이프 감소
   ↓
[웨이브 클리어 보너스] 골드 지급
   ↓
[다음 준비] ← 여기서 "어디에 뭘 더 지을까" 고민이 재미의 핵심
   ↓
웨이브 10 클리어 → 승리 / 라이프 0 → 패배
```

- 루프 1회 = 웨이브 1개. 준비 단계는 시간 무제한(플레이어가 시작 버튼을 누름), 단 웨이브 클리어 후 자동 카운트다운(예: 15초, 수치는 wave-balancer)이 표시되며 버튼으로 즉시 시작 가능.
- **왜 재미있는가**: 준비 단계의 무압박 사고와 전투 단계의 무개입 관전이 교차하며 "내 계획이 맞았는지"를 즉시 검증받는다.

### 승패 조건

- **승리**: 웨이브 10의 모든 적(보스 포함) 처치 → 승리 화면.
- **패배**: 라이프 0 → 즉시 패배 화면. 시작 라이프 20. 일반 적 누수 시 -1, 보스 누수 시 -5.
- **명확화 (v1.1)**: 승패의 최우선 판정은 라이프다. 웨이브 10에서 보스가 처치되지 않고 누수(-5)되더라도 라이프가 남아 있으면 수정을 지켜낸 것이므로 **승리**로 판정한다 (생존 = 승리, 장르 관례). AC-03은 전원 처치 경로를 검증하는 기준이며 이 판정과 모순되지 않는다.

## 3. 타워 목록 (4종 — 역할 분화)

세부 수치(비용/데미지/사거리/공속)는 wave-balancer가 확정한다. 아래 "의도"가 밸런스의 구속 조건이다.

| 타워 | 역할 | 의도 (밸런스 구속 조건) | 왜 재미있는가 | 시각 컨셉 (codex용 한 줄) |
|---|---|---|---|---|
| **애로우 타워** (Arrow Tower) | 기본 단일 딜러 | 최저가. 빠른 공속, 단일 대상, 평균 사거리. 초반 주력이지만 후반엔 업그레이드 없이는 화력 부족 | 언제 사도 손해는 아닌 안전한 선택지가 초보자의 발판이 된다 | 나무 망루 위에 석궁이 달린 소박한 감시탑, 갈색 목재 톤 |
| **캐논 타워** (Cannon Tower) | 광역(스플래시) | 중간 가격. 착탄 지점 스플래시 피해, 느린 공속. 뭉친 무리에 최강, 단일 고체력 적에게 비효율 | 고블린 러시가 한 방에 증발하는 순간의 카타르시스 | 돌 포탑 위에 뭉툭한 청동 대포, 회색 석재 + 청동 포신 |
| **프로스트 타워** (Frost Tower) | 군중 제어(CC) | 저가. 피해량은 매우 낮거나 0에 가깝고, 명중한 적에게 이동속도 슬로우(수 초). 딜 타워와 조합해야만 가치가 나온다 | 혼자서는 못 이기는 서포터 — "어디에 끼워 넣어야 딜이 극대화되나"라는 배치 퍼즐을 만든다 | 얼음 결정이 소용돌이치는 푸른 수정 첨탑, 냉기 입자 |
| **아케인 타워** (Arcane Tower) | 장거리 고단일딜 | 최고가. 최장 사거리, 높은 한 방 피해, 매우 느린 공속. 방어력 높은 적·보스 카운터. 방어력 관통 또는 마법 피해(방어 무시)로 처리 — 방식은 entity-dev 재량 | 보스전을 위해 아껴 산 비싼 타워가 제값을 하는 순간이 후반부의 보상이다 | 보라색 부유 수정이 정점에 떠 있는 어두운 마법 첨탑, 자주색 발광 |

### 공통 규칙

- 모든 타워는 3레벨 업그레이드: 업그레이드마다 주요 능력치(데미지 또는 효과) 강화. 레벨별 별도 스프라이트는 만들지 않고, 코드가 레벨 배지/색상 틴트로 표기한다 (에셋 물량 통제). **(v2 대체 — §12.1·§12.2: 레벨별 효과 차별화 + 레벨별 실스프라이트 `tower_{type}_lv1~3` 도입. 이 항목의 "배지 표기"는 v2에서 폐기)**
- 타겟팅 기본값: "경로 진행도가 가장 앞선 적" (First). 타겟 모드 전환은 MVP 제외 — 차후 확장.
- 판매: 총 투자액(건설+업그레이드)의 70% 환불.

## 4. 적 목록 (4종 + 보스 1종)

세부 수치(HP/속도/골드/등장 웨이브)는 wave-balancer가 확정한다.

| 적 | 유형 | 의도 (밸런스 구속 조건) | 왜 재미있는가 | 시각 컨셉 (codex용 한 줄) |
|---|---|---|---|---|
| **고블린** (Goblin) | 다수·저체력 | 빠른 속도, 낮은 HP, 대량 등장. 캐논(광역)의 밥 | 물량의 압박 — 화면을 채우는 무리가 광역 타워 구매를 정당화한다 | 녹색 피부에 누더기를 걸친 작은 고블린, 단검 들고 달리는 자세 |
| **오크** (Orc) | 표준 | 중간 속도, 중간 HP. 밸런스의 기준점(baseline) | 모든 수치의 비교 기준 — 플레이어가 "이 정도는 막는다"는 감을 잡게 한다 | 어깨 갑주를 걸친 근육질 회록색 오크 전사, 도끼 소지 |
| **스틸 브루트** (Steel Brute) | 고방어·저속 | 느린 속도, 높은 HP + 물리 피해 감소(방어력). 애로우로는 비효율, 아케인이 카운터 | "쏘는데 안 죽는" 답답함이 새 타워 구매(아케인)라는 해법 탐색을 강제한다 | 전신 강철 판금 갑옷의 거구 전사, 투구 사이 붉은 눈, 둔중한 걸음 |
| **와스프 러너** (Wasp Runner) | 고속·저체력 | 매우 빠른 속도, 낮은 HP. 프로스트(슬로우)가 카운터. 지상 유닛 (비행 아님 — D2) | 눈 깜짝할 새 지나가는 속도가 "슬로우를 어디에 두나"라는 동선 설계를 요구한다 | 노란-검정 줄무늬 곤충형 사족 파충류, 낮게 엎드려 질주하는 실루엣 |
| **스톤 골렘** (Stone Golem) — 보스 | 보스 | 웨이브 10 단독(또는 소수 호위 동반) 등장. 극도로 높은 HP, 느린 속도, 슬로우 저항(슬로우 효과 절반). 누수 시 라이프 -5 | 지금까지 쌓은 모든 화력의 총결산 — 체력바가 깎이는 속도로 내 빌드가 채점된다 | 이끼 낀 바위 몸통에 주황색 용암 균열이 빛나는 거대 골렘 |

### 웨이브 구성 의도 (wave-balancer 가이드)

- 웨이브 1~2: 고블린만 — 조작 학습.
- 웨이브 3~4: 오크 혼합 — 화력 증설 압박.
- 웨이브 5: 와스프 첫 등장 — 프로스트 구매 동기.
- 웨이브 6~7: 스틸 브루트 첫 등장 — 아케인 구매 동기.
- 웨이브 8~9: 전 종 혼합 러시 — 최종 점검.
- 웨이브 10: 스톤 골렘 + 소수 호위.
- 첫 실패 지점이 5~7웨이브 부근에 오도록 (첫 판 승률 체감 목표: 신중한 플레이어는 첫 판 클리어 가능, 대충 하면 진다).

## 5. 맵

- **맵 1장**: "수정 골짜기" (Crystal Valley).
- 그리드: **15 × 10 타일, 타일 64px** → 게임 필드 960×640. (HUD는 필드 위 오버레이 또는 하단 바 — ui-dev 재량, 단 경로를 가리지 않을 것.)
- 경로: 좌측 가장자리 입구 → S자 곡선 → 우측 가장자리의 수정(도착 지점). S자로 경로 길이를 확보해 타워 사거리가 경로를 2회 이상 커버하는 "명당 타일"을 만든다.
- **왜 재미있는가**: 명당(경로가 꺾여 이중 커버되는 안쪽 타일)의 존재가 배치 실력 차이를 만든다.
- 타일 종류: `경로`(건설 불가), `건설 가능 지형`(잔디), `장식/바위`(건설 불가, 소량). 입구 표식과 도착점의 수정 오브젝트 필요.
- 시각 컨셉 (codex용): 밝은 초록 잔디 타일 / 밟아 다져진 흙길 타일 / 하늘색 발광 수정 클러스터(도착점) / 어두운 동굴 입구(입구).
- 구체 경로 좌표(웨이포인트 배열)는 map-designer가 확정하고 레벨 데이터로 공유한다.

## 6. 경제

수치는 전부 wave-balancer 소관. 디렉터가 정하는 것은 구조와 의도다.

- **수입원**: ① 적 처치 보상(종별 상이) ② 웨이브 클리어 보너스 ③ 시작 골드.
- **지출처**: ① 타워 건설 ② 업그레이드. (환불 70%는 지출 완화 장치.)
- **의도**:
  - 시작 골드로 **타워 2기**(예: 애로우 2기 또는 애로우+프로스트)를 건설할 수 있어야 한다 — 첫 선택부터 조합 고민이 생기게.
  - 웨이브를 "완벽 방어"하면 다음 웨이브에 타워 1기 신설 또는 업그레이드 1회가 가능한 수준의 수입 — 성장 실감이 매 루프마다 있게.
  - 후반 웨이브 골드를 쥐고만 있으면 지는 구조 (이자 시스템 없음 — 차후 확장). 골드는 쓰라고 주는 것.
- **왜 재미있는가**: 수입이 "잘 막을수록 늘어나는" 게 아니라 고정적이므로, 차이는 오직 "어디에 썼는가"에서 나온다 — 실력이 곧 배치와 조합이 된다.

## 7. UI 요구

1. **타이틀 화면**: 게임 로고 + 시작 버튼. (조작 설명 1~3줄 포함 권장.)
2. **HUD (상시)**: 골드 / 라이프 / 현재 웨이브(예: `Wave 3/10`). 항상 보이고 게임 필드의 경로를 가리지 않는다.
3. **타워 상점**: 타워 4종 버튼(아이콘+가격). 골드 부족 시 비활성 표시. 선택 → 배치 모드 진입.
4. **배치 프리뷰**: 마우스 따라다니는 고스트 + 사거리 원. 건설 가능 타일 = 초록 계열, 불가(경로/점유/장식) = 빨강 계열. 클릭으로 확정, 우클릭 또는 ESC로 취소.
5. **타워 정보 패널**: 건설된 타워 클릭 시 — 현재 레벨/능력치, 업그레이드 버튼(비용 표시, 골드 부족·최대 레벨 시 비활성), 판매 버튼(환불액 표시), 사거리 원 표시.
6. **웨이브 컨트롤**: "웨이브 시작" 버튼 + 자동 카운트다운 표시. 웨이브 진행 중에는 다음 웨이브 정보(선택)로 대체 가능.
7. **배속 토글**: 1x / 2x. 게임 로직 속도만 바꾸고 물리 정확성을 해치지 않을 것(고정 타임스텝 전제).
8. **승리 화면**: 승리 문구 + 통계(선택: 처치 수, 남은 라이프) + 재시작 버튼.
9. **패배 화면**: 패배 문구 + 도달 웨이브 + 재시작 버튼.
10. **음소거 토글**: 사운드 on/off 버튼 1개 (BGM/SFX 분리 슬라이더는 차후 확장).

## 10. 승인 기준 체크리스트 (Acceptance Criteria)

QA와 playtester는 아래 항목을 그대로 검증한다. 최종 승인은 전 항목 통과가 조건이다.

**A. 게임 흐름**
- [ ] AC-01: 타이틀 화면이 표시되고, 시작 버튼 클릭으로 게임 화면에 진입한다.
- [ ] AC-02: "웨이브 시작" 버튼 또는 카운트다운 만료로 웨이브가 시작되고, 적들이 입구에서 생성되어 경로를 따라 도착점까지 이동한다.
- [ ] AC-03: 웨이브 10의 보스를 포함한 모든 적 처치 시 승리 화면이 표시된다.
- [ ] AC-04: 라이프가 0이 되면 즉시 패배 화면이 표시되고 도달 웨이브가 보인다.
- [ ] AC-05: 승리/패배 화면의 재시작 버튼으로 초기 상태(골드/라이프/웨이브 1)에서 다시 플레이할 수 있다.

**B. 타워**
- [ ] AC-06: 타워 4종 모두 상점에서 구매·배치할 수 있고, 배치 시 골드가 가격만큼 차감된다.
- [ ] AC-07: 골드 부족 시 해당 타워 버튼이 비활성으로 보이고 구매가 되지 않는다 (에러음 재생).
- [ ] AC-08: 경로 타일·이미 타워가 있는 타일·장식 타일에는 배치할 수 없고, 배치 프리뷰가 불가(빨강)로 표시된다.
- [ ] AC-09: 4종의 역할 차이가 실제 동작으로 확인된다 — 캐논: 착탄 지점 주변 복수 적 동시 피해 / 프로스트: 명중한 적의 이동속도가 눈에 띄게 감소 / 아케인: 스틸 브루트를 애로우보다 유의미하게 빨리 처치 / 애로우: 4종 중 최고 공속.
- [ ] AC-10: 타워 클릭 → 정보 패널에서 업그레이드가 가능하고(비용 차감, 능력치 상승), 레벨 3에서는 업그레이드 버튼이 비활성이다.
- [ ] AC-11: 판매 시 타워가 제거되고 총 투자액의 70%가 환불되며, 해당 타일에 재건설이 가능하다.

**C. 전투·경제**
- [ ] AC-12: 적 처치 시 골드가 증가하고, 웨이브 클리어 시 보너스 골드가 지급된다.
- [ ] AC-13: 일반 적이 도착점 도달 시 라이프 -1, 보스 도달 시 -5가 적용되고 해당 적은 제거된다.

### 12.1 타워 업그레이드 차별화 (v2 핵심 기획)

역할 정체성(속사 단일 / 스플래시 / CC / 장거리 저격)은 절대 불변. 세부 수치(스택 수·배율·반경·지속·틱 피해)는 전부 wave-balancer 소관 — 아래 "밸런스 구속"이 상한/하한의 의도다.

**Lv2 — 정체성 축 비대칭 강화** (새 메커니즘 없음, 성장 방향의 차별화)

| 타워 | Lv2 강화 축 | 왜 재미있는가 |
|---|---|---|
| 애로우 | 공속 위주 (데미지는 완만) | 업그레이드가 "더 세짐"이 아니라 "더 그 타워다워짐"으로 읽힌다 — 4종 공통 |
| 캐논 | 스플래시 반경 확대 위주 | 같은 명당에서 커버하는 무리가 눈에 띄게 넓어진다 |
| 프로스트 | 슬로우 지속/강도 위주 | 딜은 그대로인데 "붙잡아두는 힘"이 자란다 — 서포터 정체성 강화 |
| 아케인 | 사거리 위주 | 저격수가 더 먼 곳에서 먼저 쏘기 시작한다 |

**Lv3 — 고유 메커니즘 해금** (타워당 1개, 4종 전부 상이)

| 타워 | Lv3 메커니즘 | 정의 | 왜 재미있는가 | 밸런스 구속 |

| 애로우 | **속사 가속** (Rapid Volley) | 동일 대상 연속 명중마다 발사 간격 단축 스택(최대 N스택). 대상이 바뀌거나 죽으면 초기화 | "한 놈만 팬다"의 화력 상승이 소리·발사 리듬으로 체감된다 — 단일딜 정체성의 극대화 | 최대 가속 시에도 DPS/골드가 아케인의 단일딜 존재 이유를 침범하지 않을 것 |
| 캐논 | **화염 지대** (Burning Ground) | 착탄 지점에 수 초간 불장판 생성, 위를 지나는 적에게 틱 피해 | 순간 폭발이 "지역 장악"으로 진화 — 어디에 떨어지게 하느냐는 조준 고민이 생긴다 | 지대 틱 DPS는 직격 피해보다 낮게 — 주 딜은 여전히 착탄 |
| 프로스트 | **빙결 파동** (Frost Nova) | 명중 시 슬로우가 착탄 반경 내 주변 적에게도 확산 (광역 슬로우) | 단일 투사체 CC의 구조적 한계(밀집 열차에서 2마리만 유지 — v1 밸런스 리포트 실증)를 Lv3 투자로 해결하는 명확한 보상 | 확산 반경은 캐논 스플래시보다 작거나 같게. 보스 slowResist 규칙(§8 계약) 그대로 적용 |
| 아케인 | **과충전** (Overcharge) | 마지막 발사 후 대기 시간에 비례해 다음 발 피해 증폭 (최대 +X%) | 느린 공속이라는 약점이 "기다림 = 화력"으로 반전 — 웨이브 진입 첫 발의 알파 스트라이크 쾌감 | 최대 증폭이 상시 DPS 계산을 왜곡하지 않게 상한 설정. 저격 정체성 유지(다중 타겟화 금지) |

- 업그레이드 비용 구조(3레벨, Lv3 비용에 메커니즘 가치 반영)와 환불 70% 규칙은 v1 그대로.
- 타워 정보 패널은 Lv3 메커니즘을 1줄 설명으로 노출한다 (ui-dev — "다음 레벨에서 무엇이 해금되나"가 업그레이드 구매 동기).
- entity-dev 참고: 4개 메커니즘 모두 기존 이벤트 계약 위에서 표현 가능해야 하며, 신규 이벤트가 필요하면 architect 승인 절차를 따른다.

### 12.2 애니메이션·에셋 (v1 스타일 유지 — 밝은 카툰 판타지, 기존 팔레트·비례 준수)

1. **적 걷기 사이클**: 5종(보스 포함) × 4프레임. 시트(1행 4열) 생성 → 슬라이싱 → 아틀라스로 게임 적용. 이동 중 프레임 순환(재생 속도는 이동 속도 비례 권장 — 세부는 architect/engine). **왜 재미있는가**: 정지 그림의 미끄러짐(sliding)이 사라지는 것만으로 화면의 생동감이 한 단계 오른다.
2. **타워 레벨별 외형 3단계**: `tower_{type}_lv1~3` 12키. 레벨이 오를수록 같은 실루엣이 화려해지는 방향(구조물 증축·발광 강화) — 다른 타워로 오인되면 안 됨. Lv3는 §12.1 메커니즘을 암시하는 시각 요소 1개 포함(예: 캐논 Lv3 포신에 화염 문양, 프로스트 Lv3 첨탑에 파동 링).
3. **다각도 레퍼런스 시트**: 유닛별 다각도(정면/측면/후면)·포즈 시트. **용도 명문화**: ① 스타일 가이드 문서로 저장소에 보존 ② QA가 인게임 스프라이트와 스타일 일치(팔레트·비례·외곽선)를 대조하는 판정 기준 ③ 런타임 미사용(게임이 로드하지 않음).
4. 신규 매니페스트 키 확정·시트 규격·아틀라스 포맷은 system-architect + td-asset-pipeline v2 소관. 플레이스홀더 폴백 원칙(AC-21)은 신규 키에도 동일 적용.

### 12.3 맵 퀄리티 (시각 개선 — 게임플레이 불변, D11)

- **잔디 변형**: 2~3종 변형 타일(민무늬/클로버 점박이/들꽃 소량)을 결정적(deterministic) 패턴으로 혼합 — 단일 타일 반복의 인공적인 느낌 제거. **왜 재미있는가**: 배경이 자연스러울수록 유닛·이펙트가 더 또렷하게 읽힌다.
- **길 방향별 타일**: 직선(가로/세로)과 코너(4방향)를 구분 렌더 — 현재의 방향 무관 단일 흙길 타일 대체. 경로의 흐름이 한눈에 읽혀 신규 플레이어의 동선 파악이 빨라진다.
- **장식 밀도 상향**: 덤불·들꽃·파편 수정 등 신규 장식 2~3종 추가 배치. **구속**: 신규 장식은 경로 커버리지가 낮은 외곽 타일에 한정, 명당 타일(경로 이중 커버 인접 타일) 점유 금지, waypoints·경로 타일 집합은 v1과 동일 유지. 배치는 map-designer, 명당 침범 여부는 QA 데이터 대조.
- 시각 컨셉 (codex용): 클로버 점박이 잔디 변형 / 들꽃 무리 잔디 변형 / 굽은 흙길 코너 타일 / 낮은 초록 덤불 / 하늘색 파편 수정 조각 — 전부 v1 타일과 동일 팔레트.

### 12.4 모바일 (터치 + 반응형)

- **입력**: Pointer Events로 마우스/터치 단일 코드 경로. 터치 배치 UX: 상점 탭 → 배치 모드 → **필드 1탭 = 해당 타일 프리뷰**(고스트+사거리 원+가부 색 — 호버 대체) → **동일 타일 2탭째 = 건설 확정**, 다른 타일 탭 = 프리뷰 이동. 취소 수단 필수(모바일엔 ESC/우클릭이 없음 — 배치 모드 중 취소 버튼 노출 또는 상점 버튼 재탭). **왜 재미있는가**: 실수 배치(환불 30% 손실)가 터치 오조작으로 일어나면 부당하게 느껴진다 — 2탭 확정이 그 억울함을 없앤다.
- **세로 화면**: 세로(portrait)에서 HUD 상단 / 필드 중앙(종횡비 유지 축소) / 상점 하단으로 재배치. 스크롤·잘림 없이 전 UI 가시·조작 가능. 터치 타깃 최소 44px. 타워 정보 패널은 화면 내 클램프.
- **DPR 스케일링**: 캔버스 내부 해상도 = 논리 해상도(960×640) × min(devicePixelRatio, 2) — 고밀도 화면에서 흐릿함 제거. (v2.1 정정: 계약 §11의 상한 2 캡·논리 좌표계 불변 방식 채택)
- 데스크톱 마우스 UX(호버 프리뷰, 우클릭/ESC 취소)는 회귀 없이 유지.

## 13. v3 확장 (2026-07-08) — 5스테이지 + 종합 점수

v2까지의 게임은 "단일 맵 crystal_valley, 10웨이브, 승/패"의 완성된 한 판이다. v3은 그 한 판을 **콘텐츠 축(맵 5개)**과 **리텐션 축(점수·최고기록)** 두 방향으로 확장한다. **핵심 불변 원칙: v3은 기존 한 판의 룰(코어 루프·승패 조건·타워 4종·적 5종·업그레이드·경제 구조)을 바꾸지 않는다.** 바뀌는 것은 "그 한 판을 담는 그릇(스테이지 진행)"과 "그 한 판의 성과를 재는 자(점수)"뿐이다. 이 원칙이 v1·v2 밸런스 회귀 비용을 최소화하는 구속 조건이다.

| D13 | **기존 crystal_valley = 스테이지 1**로 편입. 신규 맵 4개(스테이지 2~5) 추가 | v1·v2 밸런스가 이 맵 기하에 완결 튜닝됨(D9-1, D11). 이를 최난도가 아닌 "학습용 1스테이지"로 재배치하면 기존 밸런스 자산을 폐기 없이 재사용하면서 5스테이지의 도입부로 자연 편입된다. crystal_valley의 완만한 S자는 튜토리얼 지형으로 최적 |
| D14 | 스테이지 진행: **순차 해금** (스테이지 N 클리어 → N+1 해금). 스테이지 1은 항상 해금. 해금 진행도는 localStorage 영속 | 난이도 점진 상승이 사용자 확정 사항이므로, 처음부터 전부 개방하면 신규 플레이어가 최난도 5스테이지에서 좌절한다. 순차 해금은 "다음 스테이지를 여는 것" 자체가 보상이 되어 리텐션을 만든다 (장르 관례 — 대부분의 캠페인형 TD가 채택) |
| D15 | 각 스테이지는 **독립된 한 판** — 자체 10웨이브, 자체 시작 골드/라이프, 클리어 시 승리 판정. 스테이지 간 골드/타워 이월 없음 | 한 판의 완결성(코어 루프 §2)을 유지. 이월 구조는 경제 밸런스를 스테이지 순서에 종속시켜 회귀 검증을 5배로 만든다. "매 스테이지는 깨끗한 새 판, 난이도만 다름"이 최소 비용으로 5배 콘텐츠를 만드는 구조 |
| D16 | **웨이브 수는 5스테이지 전부 10웨이브 고정** (D3 불변). 난이도 상승은 적 HP/수량/조합·경제 압박·맵 기하로 표현하며 웨이브 수로 표현하지 않는다 | 한 판 10~15분 리듬(D3)은 5스테이지 모두 유지되어야 세션 예측성이 산다. 웨이브 수를 늘리면 후반 스테이지가 지루하게 길어진다. 난이도는 "얼마나 오래"가 아니라 "얼마나 빡센가"로 올린다 (수치·곡선은 wave-balancer 소관) |
| D17 | 맵 5개는 **기존 타일 에셋 재조합으로 구성** — 스테이지별 신규 테마 타일셋은 MVP 제외(차후 확장). 차별화는 ① 경로 형태(기하) ② 기존 잔디/길/장식 타일의 배치 밀도·패턴 ③ (선택) 전역 색 틴트/시간대 오버레이로 표현 | 5개 테마 타일셋 신규 생성은 에셋 물량이 폭증(맵당 grass 변형·path 방향·장식 = 최소 8키 × 5 = 40키+)하고 QA 스타일 대조 부담이 5배가 된다. v2에서 이미 확보한 잔디 변형·길 방향·장식 타일(§12.3)의 재조합만으로 "서로 다른 맵 5개"라는 사용자 요구는 경로 기하 차이로 충족 가능. 신규 최소 에셋은 스테이지 선택 화면용 썸네일/배경 정도로 한정 (§13.3) |
| D18 | 점수: **종합 점수 = 처치 점수 + 웨이브 클리어 보너스 + 남은 라이프 보너스**. 스테이지별 독립 최고기록(localStorage). 전역 합산 점수/리더보드는 MVP 제외(차후 확장) | 사용자 확정 구조를 그대로 채택. 스테이지별 독립 최고기록은 "이 맵을 더 잘 깰 수 있다"는 재도전 동기(같은 맵 재플레이 시 점수 경쟁)를 만든다. 전역 리더보드는 서버가 필요하거나 로컬 한정이라 가치가 낮아 후순위 |

### 13.1 5스테이지 기획 (D13~D17)

**스테이지 = "같은 룰, 다른 지형·다른 강도의 한 판".** 5개 스테이지는 하나의 난이도 상승 곡선 위에 놓인다. 세부 수치(HP 배수·시작 골드·라이프·웨이브 조합)는 전부 wave-balancer 소관 — 아래 "난이도 의도"가 밸런스의 구속 조건(상대적 순서와 압박 방향)이다.

| # | 스테이지 명 | 테마 (색 틴트/분위기) | 경로 형태 (기하 차별화 — map-designer 확정) | 난이도 의도 (wave-balancer 구속) | 왜 재미있는가 |
|---|---|---|---|---|---|
| **1** | **수정 골짜기** (Crystal Valley) — *기존 맵* | 밝은 초록 대낮 (v1/v2 그대로) | 단일 S자 곡선 (기존 waypoints 불변 — D13·D11) | **학습용 최저 난이도.** v1/v2 튜닝 그대로. 신중하면 첫 판 클리어 가능 | 이미 검증된 "잘 만든 한 판"이 5스테이지의 자신감 있는 도입부가 된다 |
| **2** | **덤불 갈림길** (Bramble Fork) | 초록+누런 풀, 오후 톤 | 완만한 곡선이지만 **경로가 길어짐**(더 많은 코너) — 명당은 늘지만 커버할 길도 늘어남 | 1보다 소폭 상승. 적 수량·HP 완만 증가. 여전히 관대 | 길이 길어져 "어디를 이중 커버할지" 선택지가 늘어난다 — 배치 실력의 첫 시험 |
| **3** | **뒤엉킨 길** (Twin Snake) | 청록, 흐린 하늘 | **이중 굴곡(더블 S) 또는 갈래가 합류**하는 형태 — 짧고 빠른 구간과 긴 구간 혼재 | 중간 난이도. 고속 와스프·물량 압박이 빨라짐. 명당 없이는 누수 발생 | 속도가 다른 구간이 섞여 프로스트(슬로우)의 배치 위치가 진짜 고민거리가 된다 |
| **4** | **비좁은 관문** (Narrow Gate) | 어둑한 회청, 저녁 | **건설 가능 지형이 좁고 병목이 뚜렷한** 형태 — 명당은 강력하나 개수가 적음 | 높음. 제한된 타일에 최적 조합을 강제. 스틸 브루트·혼합 러시 조기 등장 | "좋은 자리가 부족하다"는 제약이 타워 조합·업그레이드 우선순위를 날카롭게 만든다 |
| **5** | **최후의 능선** (Last Ridge) | 어두운 자주+용암, 밤 | **가장 길고 복잡한 경로** — 다중 코너 + 긴 직선 사거리 구간, 보스전 압박 극대화 | 최고 난이도. 전 종 러시 강화·보스 강화(수치는 밸런서). 완성된 빌드·업그레이드 숙련 요구 | 지금까지 배운 모든 것(조합·명당·업그레이드·경제)의 총결산 — 클리어가 곧 마스터 증명 |

**진행 규칙 (D14·D15):**
- 스테이지 1은 항상 플레이 가능. 스테이지 N 클리어 시 스테이지 N+1이 해금되어 선택 화면에 개방된다.
- 해금 진행도(어디까지 깼는가)는 localStorage에 영속 저장 — 브라우저를 닫아도 유지.
- 각 스테이지는 독립된 한 판: 진입 시 시작 골드/라이프 초기화, 10웨이브 클리어 시 그 스테이지 승리(§2 승패 조건 그대로), 라이프 0 시 그 스테이지 패배. 스테이지 간 자원 이월 없음.
- 이미 클리어한 스테이지는 재플레이 가능(점수 갱신 목적) — 최고기록 경쟁이 재플레이 동기.

**스테이지 선택 화면 요구 (UI):**
- 타이틀 화면 → "게임 시작" → **스테이지 선택 화면** 진입 (기존 "시작 즉시 게임" 흐름 대체 — v3 이후 타이틀은 스테이지 선택으로 연결).
- 스테이지 5개를 카드/노드로 나열. 각 카드: 스테이지 번호·이름, 잠금 상태(잠김/해금), 해당 스테이지 최고 점수(있으면), 클리어 여부 표식.
- 잠긴 스테이지는 선택 불가(자물쇠 표시) + 해금 조건 안내("스테이지 N 클리어 시 해금").
- 해금된 스테이지 선택 → 그 맵으로 게임 시작. 게임 중/결과 화면에서 스테이지 선택으로 돌아가는 경로 필요.
- 세부 시각·레이아웃은 ui-dev(모바일 세로 대응 §12.4 준수 — 5카드가 세로에서 스크롤/잘림 없이).

### 13.2 종합 점수 시스템 (D18)

**점수 = 한 판의 성과를 하나의 숫자로 압축한 것.** 배점 세부 수치(적 종류별 점수, 웨이브 보너스 값, 라이프 보너스 계수)는 wave-balancer 소관 — 아래는 "무엇에 점수를 주는가"의 구조와 의도.

**배점 구조 (3요소 합산):**
| 요소 | 의도 (밸런서 구속) | 왜 재미있는가 |
|---|---|---|
| **① 처치 점수** | 적 종류별 차등 — 처치가 어려운 적(고체력·보스)일수록 높게. 보스가 단일 최고 배점 | "무엇을 잡았는가"가 점수에 반영 — 강적 처치의 보상감 |
| **② 웨이브 클리어 보너스** | 웨이브를 넘길 때마다 가산. 후반 웨이브일수록 크게(선택) | "얼마나 나아갔는가" — 중도 패배해도 도달 웨이브만큼 점수 인정, 재도전 시 진전 체감 |
| **③ 남은 라이프 보너스** | 클리어 시 남은 라이프 × 계수. 무피해 클리어가 최고점 | "얼마나 완벽했는가" — 이미 이긴 판을 "더 잘 이기려는" 재플레이 동기의 핵심 |

- **점수 자체 규칙**: 판매·업그레이드는 점수에 영향 없음(경제 선택은 클리어/라이프 보너스로 간접 반영). 배속 사용은 점수 페널티 없음(접근성 — 배속은 편의지 치트가 아님). 점수는 스테이지 독립(스테이지마다 별도 최고기록).
- **왜 이 구조인가**: 세 요소가 각각 "전투 성과 / 진행도 / 방어 완성도"를 담당해, 단일 지표로 압축되면서도 "다르게 잘하는 법"이 여럿 생긴다 — 같은 클리어여도 무피해 클리어가 더 높은 점수가 되어 재플레이 목표가 명확해진다.

**HUD 표시 (ui-dev):**
- 게임 중 HUD에 현재 누적 점수를 실시간 표시(골드/라이프/웨이브 옆). 처치·웨이브 클리어 시 즉시 가산 반영. 경로 가리지 않음(§7-2 원칙).
- (선택) 점수 증가 시 짧은 플로팅/팝 연출 — fx-dev 재량, 필수 아님.

**결과 화면 (승리/패배 공통 확장 — ui-dev):**
- 이번 판 최종 점수 표시 (요소별 분해 표시 권장: 처치 X + 웨이브 Y + 라이프 Z = 합계).
- 해당 스테이지 최고기록 표시. **이번 점수가 최고기록 경신 시 "신기록!" 연출**(강조 텍스트·이펙트·징글 — fx-dev/audio-dev 재량).
- 패배 화면도 점수 표시(도달 웨이브까지의 누적 점수) — 중도 패배도 점수가 남아 재도전 동기.

**최고기록 저장 (engine/architect):**
- **스테이지별 독립 최고기록**을 localStorage에 저장. 저장 범위: 스테이지 해금 진행도 + 스테이지 5개 각각의 최고 점수. (게임 중 상태 세이브/이어하기는 범위 아님 — §11 유지.)
- 저장 스키마 키·버전 관리는 system-architect 확정. 손상/부재 시 초기값(1스테이지만 해금, 최고기록 0)으로 폴백 — 크래시 금지.

### 13.3 에셋 추가 필요 여부 (D17 판정)

**결론: 맵 타일 신규 테마셋은 불필요. 최소 신규 에셋만 추가.**

- **맵 5개 타일**: v2에서 확보한 에셋(잔디 변형 2~3종, 길 직선/코너, 장식 2~3종)의 **재조합 + 경로 기하 차이 + 전역 색 틴트**로 충분(D17). 스테이지 2~5의 차별화는 코드/데이터(waypoints·타일 배치·틴트 오버레이)로 달성하며 신규 스프라이트를 요구하지 않는다. → **asset-artist 추가 작업 없음(맵 타일 한정).**
- **신규 에셋 최소 목록 (asset-artist — 판단은 system-architect가 매니페스트 확정 시 최종 결정):**
  - (권장) 스테이지 선택 화면 배경/썸네일 — 5스테이지를 시각 구분할 카드용. 단, 기존 타일로 조합한 미니맵 프리뷰로 대체 가능하면 신규 생성 생략 가능(비용 절감 우선).
  - (선택) "신기록!" 연출용 뱃지/별 아이콘 1종 — 없으면 fx 텍스트+파티클로 대체.
- **폴백 원칙(AC-21) 불변**: 신규 에셋이 있든 없든 도형 플레이스홀더로 게임은 실행 가능해야 한다.

### 13.4 v3 범위

**포함**: §13.1 5스테이지(기존=1스테이지 + 신규 맵 4개, 순차 해금, 스테이지 선택 화면) / §13.2 종합 점수(3요소 합산, HUD 실시간, 결과 화면 분해·최고기록·신기록 연출) / 스테이지별 최고기록·해금 진행도 localStorage 영속 / (최소) 스테이지 선택 UI 관련 에셋.

## 14. v4 확장 (2026-07-19) — 비주얼 업그레이드 (= "GDD v4 개정")

v1~v3으로 게임은 "5스테이지 · 종합 점수 · 업그레이드 차별화"까지 완결된 한 판이다. v4는 **오직 시청각 밀도**만 끌어올린다 — 룰·수치·데이터는 한 글자도 바꾸지 않는다. 사용자 요구는 카툰 스틸 스프라이트를 "살아 움직이는 3D 룩 화면"으로 전환하는 6개 축이다. **v4의 절대 원칙: 비주얼 전용. 밸런스·게임플레이·경로 기하·점수·저장 스키마는 불변(회귀 금지 §14.6).** 이 원칙이 v1~v3의 방대한 회귀 자산(sim 45항목·AC 48건)을 재튜닝 없이 보존하는 구속 조건이다. **왜 재미있는가**: TD는 조작 없이 관전하는 시간이 길다 — 그 시간을 "정지 그림 감상"에서 "살아 숨쉬는 전장 관전"으로 바꾸는 것이 v4의 재미 가설이다.

### 14.2 시각 컨셉 지시

#### 14.2.1 타워 4종 — idle / attack 모션 컨셉 (asset-artist 프롬프트 근거)

역할 정체성(§3)과 Lv3 메커니즘(§12.1)이 모션에서 읽히도록 지시한다. 조명은 전 시트 상단좌측 고정, 레벨 간 실루엣·색계 일관(오인 방지).

| 타워 (역할) | idle 루프 (0행 4프레임) | attack one-shot (1행 4프레임) | Lv3 시트 암시 요소 |
|---|---|---|---|
| **애로우** (속사 단일) | 망루 위 석궁이 좌우로 미세하게 조준 스윙 + 나무 깃발 펄럭임 | 시위 당김(장전)→발사 섬광→반동으로 뒤로 젖힘→복귀 (빠른 리듬) | 속사 가속(Rapid Volley) — 석궁에 예비 화살통·발사 잔상(속도감) |
| **캐논** (스플래시) | 포신 미세 상하 흔들림 + 포구에서 옅은 연기 실오라기 + 청동 표면 하이라이트 | 포신 후퇴 장전→포구 대폭발 화염+연기 구름→강한 반동→포신 복귀 (묵직) | 화염 지대(Burning Ground) — 포신에 잔불·화염 문양(§12.2-2 계승) |
| **프로스트** (CC) | 얼음 결정 천천히 회전 + 냉기 입자 상승 + 내부 푸른 발광 맥동 | 냉기 응축(결정 밝아짐)→냉기 구슬 방출 섬광→결정 잠시 흐려짐→복귀 | 빙결 파동(Frost Nova) — 첨탑에 파동 링 문양(§12.2-2 계승) |
| **아케인** (장거리 저격) | 부유 수정이 공중에서 천천히 회전·상하 부유 + 자주색 룬 발광 맥동 | 마력 충전(수정 강발광·룬 점등)→아케인 볼트 발사 섬광→수정 잠시 암전→복귀 (느리고 강한 알파) | 과충전(Overcharge) — 정점 수정에 응축된 에너지 코어·오버로드 글로우 |

- **왜 이 방향인가**: idle은 "타워마다 다르게 살아있음"을, attack은 "무엇을 쏘는가"를 모션으로 각인시킨다. 애로우의 빠른 리듬 vs 아케인의 느린 알파는 §12.1 정체성을 몸짓으로 재확인시킨다.

| 1 | 수정 골짜기 (밝은 초록 대낮) | 풀 기본+변형(클로버/꽃) + 길 방향 타일 + 수정 오브젝트 — v3 배치 유지, 3D 룩 재생성 | 경로(다진 흙길), 수정 클러스터 |
| 2 | 덤불 갈림길 (초록+누런 풀, 오후) | 풀 + **흙/모래 전이(grass→dirt edge)** + 덤불 장식 밀도↑ | 경로, 바위·덤불 군집 |
| 3 | 뒤엉킨 길 (청록, 흐린 하늘) | 풀 + **물 패밀리(기본+물가 전이 4방향)** — 갈래/합류 지형에 물길 암시 | 경로, **물(건설 불가 명확)** |
| 4 | 비좁은 관문 (어둑한 회청, 저녁) | 풀 + **절벽/바위 지대 융기 타일** — 병목의 "벽"을 지형으로 표현 | 경로, **절벽/바위 지대(융기·그림자로 건설 불가 명확)** |
| 5 | 최후의 능선 (어두운 자주+용암, 밤) | 절벽 + **용암/균열 지대(emissive)** + 어두운 전역 색 틴트 | 경로, **용암·절벽(발광 위험 신호)** |

- **왜 이 방향인가**: 각 스테이지가 "다른 지형처럼" 보이면서도(콘텐츠 체감) 경로 기하는 v3 그대로라 밸런스가 보존된다. 건설 불가 지형(물·절벽·용암)은 융기·발광·질감으로 "여긴 못 짓는다"가 프리뷰 없이 읽히게 한다(AC-56).

### 14.4 애니메이션 장식 선정 (terrain-anim 레이어 1.5)

코드 규약상 맵당 소수(최대 3종)로 제한한다. v4에서 움직일 장식을 아래로 확정한다:

| 선정 | 대상 키 | 모션 | 적용 맵 | 우선순위 |
|---|---|---|---|---|
| ① **목표 수정 반짝임** | `goal_crystal` | 광채 맥동·내부 발광 순환 (idle 1행×4열) | 전 5맵(목표물) | **필수** — 게임 목표물이라 시선 집중이 정당 |
| ② **덤불 흔들림** | `deco_bush` | 바람에 잎이 좌우로 흔들림 | 덤불 갈림길 등 자연 테마 | **필수** — 자연 생동감 |
| ③ **파편 수정 반짝임** | `deco_crystal_shard` | 표면 글린트·미세 회전 | 수정 골짜기·최후의 능선 | 선택 (테마 강화) |

- **물 글린트(뒤엉킨 길)는 terrain-anim 시트가 아니라 fx 코드 오버레이(글린트 파티클)로 처리** — 코드 규약 지형 레이어: seamless 물 애니메이션 타일은 실패율이 높다. fx-dev 소관.
- **맵당 예산**: 한 맵의 terrain-anim 장식은 3종을 넘지 않는다(수정 골짜기 = 목표 수정 + 파편 수정 + 덤불 = 최대 3, 상한). 나머지 장식(바위 등)은 정적 배경 캐시(레이어 1)에 굽는다.

### 14.5 매니페스트·계약 영향 (system-architect 확정 사항 — 디렉터는 의도만 명시)

- **타워 12키 승격**: `tower_{type}_lv1~3` 문자열 → `{img, atlas}` 객체(시퀀스 `idle:0-3, attack:4-7`, 2행×4열, frame 128). 계약·매니페스트 개정은 architect 승인 후. 로더 시퀀스 폴백(요청 시퀀스 부재→첫 시퀀스 강등)은 정적 폴백에서도 안전해야 한다.
- **terrain-anim 신규 애니메이션 키**: `goal_crystal`·`deco_bush`(필수), `deco_crystal_shard`(선택)를 애니메이션 시트({img,atlas}, 1행×4열 idle)로 승격 또는 신규 키 추가 — 정적 폴백 유지.
- **신규 타일 패밀리 키**: 물(기본+물가 전이 4방향), 흙/모래 전이, 절벽/바위 지대, 용암/균열 계열 — 필요 최소 키 수는 architect가 확정(파이프라인 §7 패밀리 기준). 폴백 접두사 규칙(§manifest) 준수.
- **이벤트 계약**: 신규 이벤트 0. 진화 연출은 기존 `tower:upgraded` 재사용(D21). fx 시그니처 이펙트는 기존 발사/명중 이벤트 위에서 표현(신규 이벤트 필요 시 architect 승인 절차).
- **폴백 원칙(AC-21) 불변**: 신규 에셋이 있든 없든 도형 플레이스홀더/시퀀스 강등으로 게임이 실행 가능해야 한다.
