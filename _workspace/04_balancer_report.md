## 1. 검증 방법의 진화 (D9-1 사후 분석 — 중요)

### v1: 추상 DPS 예산 모델의 실패

초기 검증은 td-balance-design §3의 부등식(유효 DPS × 전 경로 노출 시간 ≥ EHP)으로 수행했고 목표를 통과했다. 그러나 qa-engineer의 회차 9 실엔진 자동 플레이(D9-1)에서 3가지 전략 모두 패배 — 추상 모델이 **약 2배 낙관**임이 실측으로 드러났다.

**원인 3가지 (전투 로직 결함 아님 — combat/enemy/projectile 코드는 계약 §8 그대로임을 코드 리뷰로 확인):**

1. **전 경로 노출 가정**: 모델은 타워가 적의 전체 이동 시간 동안 사격한다고 가정하지만, 실제 타워는 자기 사거리 안에 적이 있는 동안만 사격한다 (사거리 160 타워의 경로 체류 커버는 25~45%).
2. **슬로우 노출 +40%의 과대 적용**: frost는 단일 대상 투사체라 9마리 와스프 열차에서 동시에 ~2마리만 슬로우 유지 가능. 모델은 전 개체에 +40%를 적용했다. (W5 누수 7의 직접 원인)
3. **First 집중 사격의 오버킬**: 모든 타워가 최전방 1마리에 사격을 집중해 초과 피해와 비행 중 투사체가 낭비된다.

### v2: 실엔진 자동 플레이가 권위 판정

sim.mjs를 개정해 **실엔진(systems/combat·waves + entities, 수치 주입 없음)을 그대로 구동하는 봇 3종**을 난이도 판정의 권위로 삼았다. 시뮬 = 실플레이 로직이므로 이 괴리는 구조적으로 재발하지 않는다. 추상 모델은 "이론 상한" 참고 출력으로 강등.

| 봇 | 정의 | 게이트 |
|---|---|---|
| A 산개 무전략 | 아무 데나 배치, 업그레이드 없음 | 웨이브 5~7 실패 |
| B 도배 무업글 | 명당 배치, 조합/업그레이드 없음 | (참고 지표) |
| C 킬존 최적 | 커버리지 기하 + 업그레이드 인터리브 (qa v3 큐) | 클리어 + 잔여 30~70% |

봇은 판매·재배치·타이밍 조작이 없으므로 "신중한 플레이어"의 **하한**이다. 사람은 이보다 잘한다 → C가 50% 잔여로 클리어하면 실제 킬존 플레이어는 그 이상의 여유를 갖는다.

## 2. 공식

```
성장:   hpMultiplier(n) = 1.18^(n-1)   // 스킬 권장 구간(1.18~1.25) 하한 — 실엔진 보정 결과
보상:   적별 고정 reward + count 성장  // HP 성장보다 완만 → 후반 골드 압박 유지
경로:   1728px — levels.js 실측 (웨이포인트 8개, 27스텝 × 64px)
효율:   arrow Lv1 = 0.229 DPS/골드 기준. cannon 밀집 1.2배 / frost는 DPS 제외(CC 가치) / arcane 0.78 (사거리+armor 무시 프리미엄)
업그레이드: 비용 = 건설비 × 0.8 / 1.2 (Lv2/Lv3), 데미지 ≈ ×1.6/단계
```

## 6. 남은 확인 사항 (playtester)

sim v2는 실엔진이지만 봇이다. 사람 기준 확인 요청:

1. **킬존 플레이 체감**: 봇 C가 50% 잔여 — 사람이 판매/재배치까지 쓰면 70%를 넘겨 "너무 쉬움"이 될 수 있다. 그 경우 W8~10 hpMultiplier만 +8~10% 소폭 상향 (후반 스파이크, 조기 웨이브는 유지).
2. **W2 체감**: 무전략 봇도 W2에서 6 누수 — 사람 초심자가 첫 판에 여기서 라이프 30%를 잃으면 가혹할 수 있음. 그 경우 W2 goblin 10→8.
3. **보스전 길이**: golem 도보 48초(1x). 지루하면 speed 36→42 + hp 소폭 상향으로 총 EHP 보존.
4. 배속 2x에서의 판정 정상 여부는 AC-17 (qa 소관).

체감 리포트가 시뮬과 어긋나면 시뮬(봇 전략)을 먼저 의심한다. 문의: wave-balancer.
