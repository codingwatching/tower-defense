# TD Balance Design — 밸런스 설계 방법론

감이 아니라 모델로 수치를 만든다. 모델 → 데이터 모듈 → 시뮬 검증 → 실플레이 보정의 순서를 지킨다.

## 1. 타워 아키타입 — 역할 분화가 먼저

4종 타워는 DPS가 아니라 **역할**로 구분한다. 역할이 겹치면 하나는 죽은 타워다.

| 아키타입 | 역할 | 수치 특성 |
|---|---|---|
| 저격(Arrow) | 단일 대상 원거리 | 긴 사거리, 중간 DPS, 저비용 — 기준 타워 |
| 포격(Cannon) | 스플래시 광역 | 짧은 사거리, 높은 발당 데미지, 느린 공속, 물량전 카운터 |
| 감속(Frost) | 군중 제어 | 데미지 거의 0, 슬로우 40~50%, 킬존 형성용 |
| 속사(Gatling) | 근접 고DPS | 최단 사거리, 최고 DPS, 고비용, 중갑 카운터 |

**비용 효율 규칙:** `DPS ÷ 비용`을 기준 타워(저격) 1.0으로 놓고, 제약이 큰 타워일수록 효율을 높인다 (속사 1.2~1.3, 포격은 다중 명중 기대값 포함 1.1~1.2). 감속은 DPS 효율에서 제외 — 가치는 "적이 킬존에 머무는 시간 증가분"이다.

## 2. 적 성장 곡선

```
HP(wave n)    = HP_base × growth^(n-1)     // growth 1.18~1.25
count(wave n) = count_base + floor(n × 1.5)
reward(n)     = reward_base × growth^((n-1) × 0.6)  // HP보다 완만하게
```

- **보상 성장 < HP 성장**이 핵심이다: 후반으로 갈수록 골드가 상대적으로 부족해져 업그레이드 선택이 절박해진다. 같은 비율로 키우면 후반이 자동으로 쉬워진다.
- 적 4종 역할: 기본(기준) / 고속(HP 60%, 속도 170% — 저격 카운터) / 중갑(HP 250%, 속도 60%, 방어 — 속사 강요) / 물량(HP 40%, 밀집 스폰 — 포격 강요). 보스는 5·10웨이브에 배치, HP는 해당 웨이브 총 EHP의 40~60%.

## 3. DPS 예산 검증 (핵심 부등식)

웨이브 n을 막을 수 있는가는 다음으로 근사한다:

```
필요 조건:  플레이어 총 DPS × 노출 시간(경로 길이 ÷ 적 속도) ≥ 웨이브 총 EHP
플레이어 총 DPS ≈ 누적 골드 수입 × 평균 효율(DPS/gold) × 투자율(~0.9)
```

**난이도 목표 (GDD 의도의 수치화):**
- 무전략(타워 아무 데나) 배치 = 유효 DPS 60% 가정 → 5~7웨이브에서 부등식이 깨져야 한다
- 킬존 활용(사거리 겹침 + 감속) = 유효 DPS 95% + 감속의 노출 시간 +40% → 최종 웨이브를 라이프 30~70% 여유로 클리어

## 4. 경제 상수 기본값

| 상수 | 기본값 | 근거 |
|---|---|---|
| 시작 골드 | 기준 타워 2기 + 여유 20% | 첫 웨이브 전 의미 있는 선택 2번 |
| 시작 라이프 | 20 (기본 적 통과 = 1, 보스 = 5) | 초반 실수 3~4회 허용 |
| 판매 환급률 | 70% | 재배치를 허용하되 무료 리셋은 금지 |
| 업그레이드 비용 | 건설비 × 0.8 / 단계, 효과 +60%/단계 | 신규 건설과 업그레이드가 팽팽한 선택이 되도록 |
| 웨이브 클리어 보너스 | 다음 웨이브 예상 수입의 15% | 조기 웨이브 소환 인센티브의 재원 |

## 5. 헤드리스 시뮬레이션

`scripts/sim.mjs`는 `src/data/*.js`를 import해서 (렌더링 없이) 검증한다:

```bash
node scripts/sim.mjs
```

최소 검증 항목: ① 웨이브별 총 EHP vs 누적 가용 DPS 표 출력 ② 무전략/킬존 두 시나리오의 실패 웨이브 ③ 골드 수지 (파산 웨이브 없는지). 시뮬은 근사 모델이다 — 시뮬 통과 후에도 playtester의 체감 리포트가 최종 판정이다.

## 6. 튜닝 이터레이션 규칙

- 한 번에 바꾸는 축은 1~2개로 제한한다 (성장률과 보상을 동시에 크게 바꾸면 원인 추적 불가).
- playtester 체감 → 수치 대응 표:

| 체감 | 우선 조정 |
|---|---|
| "초반부터 못 막겠다" | 시작 골드 ↑ 또는 1~3웨이브 count ↓ |
| "중반에 급격히 무너진다" | growth ↓ 0.02~0.03 단위 |
| "후반이 지루하게 쉽다" | 보상 성장 지수 ↓ 또는 growth ↑ |
| "한 타워만 도배하게 된다" | 그 타워 비용 효율 ↓ 10%, 카운터 적 비중 ↑ |
| "감속 타워를 안 쓴다" | 고속 적 비중 ↑ (감속 수치보다 필요성을 만든다) |
